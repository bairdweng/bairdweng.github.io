异常捕获方法总结

2021-04-30 1 min read
1. 异常类型

  1. NSException 通常是Objetive-C的异常，比较方便能处理，类似数组越界，nill的处理。

     // 获取堆栈信息
 NSArray *callStack = [exception callStackSymbols];

  2. signal的处理

    static const int g_fatalSignals[] =
{
 SIGABRT,//调用abort生成的信号，有可能是NSException也有可能是Mach
 SIGBUS,//非法地址
 SIGFPE,//算术运算错误
 SIGILL,//执行非法指令
 SIGPIPE,//进程间通信产生，通信管道破裂
 SIGSEGV,//非法访问地址，比如野指针，通常是对象的异常释放，指针未清空
 SIGSYS,//非法的系统调用
 SIGTRAP,//断点指令，一般出现在debug调试时
};
